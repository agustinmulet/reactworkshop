**Algo para tener en cuenta cuando iteramos:** Si vemos la consola, React nos
dice que cada hijo en un array o iterador debe tener una prop √∫nica que sea
`key` (`clave`), tiene que ser un valor √∫nico que no se repita dentro del array.
**Pasar el √≠ndice como key se considera malas pr√°cticas** ya que si eliminamos o
movemos datos dentro del array, ese √≠ndice va cambiando, algo similar puede
pasar utilizando el id como key _si no lo manejamos bien_. En nuestro caso la
`key` puede ser el d√≠a de la semana que sabemos que va a ser √∫nico, entonces
pasamos eso como prop a lo que se itera, nos quedar√≠a:

`<div className="card" key={dia.valid_date}>`

Tambi√©n vemos que las fechas m√°xima y m√≠nima desde la API nos la proveen como un
n√∫mero con decimal, al utilizar `parseInt()` obtenemos la parte entera del
n√∫mero √∫nicamente (en lo personal me gusta m√°s un n√∫mero entero).
